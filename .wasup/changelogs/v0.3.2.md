## [v0.3.2] - 2026-05-07

### Fixed

#### Identity Parsing — Whitespace Trimming (M1)

- **`Identity.parse()`** and **`parseTagger()`** now trim trailing whitespace from the name field before splitting on `<` ([`commit.zig`](src/object/commit.zig))
  - Root cause of **18 test failures**: trailing space in name produced empty timestamp token via `splitScalar`
  - Fixes: 7 commit parse + 8 tag parse + 3 identity unit tests
  - Skip empty tokens when parsing timestamp/timezone fields

#### OID Data & Hex Operations (M2)

- Corrected **OID test data lengths**: replaced 39-char and 41-char hex strings with valid 40-char OIDs ([`oid.zig`](src/object/oid.zig))
  - Fixed odd-length short-hex in `zero-from-short` test (7 → 40 chars)
  - Fixed invalid-length test to use empty string instead of odd-length `'abc'`
  - Fixed `short()` comparison to slice first N bytes of 40-byte result
  - Replaced `catch unreachable` in **`blob.oid()`** with graceful error return (**+2 ABRT crashes fixed**)
- Resolved **`@memcpy` length mismatches** across blob and OID modules:
  - [`blob.zig`](src/object/blob.zig): destination slice now matches source length
  - [`oid.zig`](src/object/oid.zig): `short()` @memcpy uses equal-length slices; corrected short-from-hex test for right-padded hex layout
- **Error type corrections** in [`commit.zig`](src/object/commit.zig): expect `InvalidIdentity` for missing timezone (not `InvalidTimezone`); fuzz tests now expect `MissingAuthor` / `NotATree`

#### Object Parse Error Alignment (M3)

- Aligned **`Object.parse()`** error types with actual Zig `parseInt` behavior ([`object.zig`](src/object/object.zig)):
  - Negative size → `Overflow` (not `InvalidObjectSize`)
  - Float/hex size → `InvalidCharacter` (not `InvalidObjectSize`)
  - Trailing spaces in type → empty size_str → `InvalidCharacter`
  - Tab delimiter → no space split → `InvalidObjectFormat`
  - Fixed `'size zero with content'`: blob header + content IS valid
  - Fixed `'extra null bytes'`: content len ≠ size → `InvalidObjectSize`
  - Fixed `computeHeaderSize`: returns fixed max (16), not variable
  - **9 object_fuzz_test failures resolved**

#### Tree Parse Memory Safety (M4)

- **`Tree.parse()`** now accepts an `allocator` parameter instead of hardcoding `std.testing.allocator` ([`tree.zig`](src/object/tree.zig))
  - Updated all 19 call sites (tests + production code)
  - Production callers use `self.allocator` (`ls_tree`, `checkout/tree`)
  - Eliminates memory leak root cause from test allocator leaking into lib code
  - **11 leaks + 2 failures fixed**

#### CI

- Critical **Windows compile error** resolved ([`build.zig`](build.zig))

### Changed

#### Test Quality Improvements (S1–S3)

- **ReflogManager tests** ([`reflog.zig`](src/ref/reflog.zig)): replaced placeholder `expect(true)` with real validation:
  - `init`: verifies allocator field is set correctly
  - `Timezone.formatZoned`: tests `+0000`, `-0400`, `+0230` output
  - `Timezone.parse`: roundtrip for UTC, negative, half-hour offsets
  - `Timezone.parse` rejects: missing sign, wrong length, non-numeric
  - `Timezone.toArray`: format→parse roundtrip preserves offset
  - `ReflogEntry struct`: validates all fields stored correctly
- **MergeAnalyzer tests** ([`analyze.zig`](src/merge/analyze.zig)): replaced `dummyGetCommit` null-stubs with real mock commits:
  - Fast-forward detection: base→tip returns `is_fast_forward=true`
  - Divergent branches: left/right from base returns `is_normal=true`
  - `canMerge`: related branches with common ancestor returns true
- **FastForwardChecker tests** ([`fast_forward.zig`](src/merge/fast_forward.zig)): replaced `dummyGetCommit` stubs with real mock commits:
  - Linear history: base→mid→tip detects FF with 2 commits moved
  - Divergent history: left/right from common base rejects FF
  - Up-to-date: same OID returns `can_ff=true` with 0 moves
  - `getMergeBase`: finds correct common ancestor for divergent branches
