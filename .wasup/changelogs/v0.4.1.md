## [v0.4.1] - 2026-05-09

### Fixed

#### Critical Memory Safety — Object.parse() Invalid Free (`915295b`)

**Root cause**: [`Object.parse()`](src/object/object.zig#L98) returned a borrowed slice into the input buffer instead of owning its data. When callers freed the input buffer before the `Object`, subsequent access to `Object.data` triggered an **"Invalid free" panic** — a use-after-free that crashed 37 tests across object/blob/commit/tag/tree subsystems.

**Fix**: Added explicit `allocator` parameter to `Object.parse()`. The function now duplicates content via `allocator.alloc(u8, content.len)` + `@memcpy(owned_data, content)`, giving each `Object` true ownership of its data with a well-defined lifetime tied to the allocator.

This is a **breaking API change** for direct callers of `Object.parse()`. All downstream call sites have been updated in this release.

#### Upgrade Notes — Breaking API Change

If you have custom code calling these functions directly, add the allocator parameter:

```zig
// Before (v0.4.0):
const obj = try Object.parse(raw_data);

// After (v0.4.1+):
const obj = try Object.parse(raw_data, allocator);
```

Same pattern applies to:
- `Blob.parse(data)` → `Blob.parse(data, allocator)`
- `Tag.parse(data)` → `Tag.parse(data, allocator)`
- `Commit.parse(allocator, data)` — unchanged (already had leading allocator)
- `Tree.parse(allocator, data)` — unchanged (already had leading allocator)
