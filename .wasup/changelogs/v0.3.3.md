## [v0.3.3] - 2026-05-07

### Changed

#### Shared Object I/O Layer — `object/io.zig` (M1 + M4)

- **New module** [`object/io.zig`](src/object/io.zig) centralizes object reading and writing:
  - `readObject(git_dir, io, allocator, oid)` — unified OID→hex path→readFileAlloc→Zlib.decompress pipeline
  - `readObjectOpt(git_dir, io, allocator, oid_hex)` — optional-return variant for describe/blame use cases
  - `writeLooseObject(git_dir, io, allocator, data)` — SHA1→dir creation→zlib.compress→write pipeline
- **15 files migrated** from inline implementations to shared calls:
  [`cli/commit.zig`](src/cli/commit.zig), [`cli/log.zig`](src/cli/log.zig), [`cli/show.zig`](src/cli/show.zig), [`cli/cat_file.zig`](src/cli/cat_file.zig), [`cli/ls_tree.zig`](src/cli/ls_tree.zig), [`cli/cherry_pick.zig`](src/cli/cherry_pick.zig), [`cli/revert.zig`](src/cli/revert.zig), [`cli/format_patch.zig`](src/cli/format_patch.zig), [`cli/archive.zig`](src/cli/archive.zig), [`describe/describe.zig`](src/describe/describe.zig), [`stash/apply.zig`](src/stash/apply.zig), [`clean/gc.zig`](src/clean/gc.zig), [`reset/hard.zig`](src/reset/hard.zig), [`reset/restore_source.zig`](src/reset/restore_source.zig), [`cli/filter_repo.zig`](src/cli/filter_repo.zig)
- **~500 lines of duplication eliminated**

#### Unified HEAD Resolution — `commit/head.zig` (M2)

- **New function** [`resolveHeadOid(git_dir, io, allocator) ?OID`](src/commit/head.zig) in existing head module:
  - Reads HEAD file → trims whitespace → follows symbolic `ref: ` refs → parses detached hex OID
  - Returns `null` for empty/unresolvable repos (consistent null-safety)
- **9 files migrated** from inline HEAD parsing:
  [`cli/commit.zig`](src/cli/commit.zig), [`cli/log.zig`](src/cli/log.zig), [`cli/format_patch.zig`](src/cli/format_patch.zig), [`cli/archive.zig`](src/cli/archive.zig), [`describe/describe.zig`](src/describe/describe.zig), [`blame/blame.zig`](src/blame/blame.zig), [`cli/rev_list.zig`](src/cli/rev_list.zig), [`cli/name_rev.zig`](src/cli/name_rev.zig)
- Each consumer adapts the nullable return to its own error type (`error.NoHead`, `error.InvalidOid`, etc.)
- **~200 lines of duplication eliminated**, error handling now consistent across all consumers

#### Shared Test Helpers — `testing/mock.zig` (M3)

- **New module** [`testing/mock.zig`](src/testing/mock.zig) with reusable test utilities:
  - `makeMockCommit(oid_hex, parent_hexs)` — factory for mock Commit objects (author/committer "a <a@b>", timestamp 0)
- **2 merge test files updated**: [`merge/fast_forward.zig`](src/merge/fast_forward.zig) and [`merge/analyze.zig`](src/merge/analyze.zig)
- All 10 call sites changed from local `makeMockCommit()` to `mock.makeMockCommit()`
- **~30 lines eliminated**

#### Compatibility Test Refactoring — `final/compat.zig` (S1)

- **New helper** `setupRepo(name)` consolidates the repeated 5-step pattern:
  cleanupDir → makeDir → git init → hoz init → return path
- **13 `run*()` methods simplified**: runInit, runAdd, runCommit, runBranch, runCheckout, runMerge, runRebase, runStash, runTag, runLog, runDiff, runShow, runBlame, runBisect, runWorktree, runRemote
- Each method reduced from ~20 lines of boilerplate to a single `setupRepo()` call
- **~130 lines eliminated**

#### Commit Tree Builder — `cli/commit.zig` (S2)

- **New method** `writeEmptyTree(git_dir)` extracts the `"tree 0\x00"` → serialize → write pattern
- Replaces **4 identical blocks** inside [`writeTree()`](src/cli/commit.zig) (index missing / parse fail / empty entries / build failure)
- Function reduced from 41 lines to 26 lines
- **~28 lines eliminated**
