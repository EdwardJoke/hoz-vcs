# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [v0.5.6] - 2026-05-12

### Added
- **install.sh download progress bar** — Replace silent `-fsSL` / `-q` flags with `--progress-bar` for curl and `--progress=bar:force:noscroll` for wget, showing real-time download progress during installation (`36860f5`)
- **Codeberg download support** — New `--codeberg` flag forces download from Codeberg; when omitted, installer tries GitHub first and automatically falls back to Codeberg on failure (`4766ab7`)

### Fixed
- **Silent error swallowing in critical I/O paths** (M1) — Replaced `catch {}` and `catch return;` with proper error propagation or `std.log.warn` logging across 8 modules:
  - `ref/branch.zig` — config file writes now propagate errors instead of silently discarding failures
  - `cli/add.zig` — index and object write errors now surface to caller
  - `index/index.zig` — `getUnmergedEntries` no longer swallows parse errors
  - `network/ssh_native.zig` — SSH channel read failures now logged with error context
  - `network/pack_recv.zig` — pack object storage errors now propagated
  - `network/resilience.zig` — temp file deletion failures now logged as warnings
  - `network/transport.zig` — ref enumeration errors now logged
  - `cli/log.zig` — ref walking errors now surfaced via logging
  - `object/reader.zig` — test error swallowing fixed
- **openDirAbsolute with relative paths** (M2) — Worktree modules and loose object iterator now resolve relative paths to absolute before calling `std.fs.openDirAbsolute`, which requires absolute paths:
  - `worktree/remove.zig` — added `resolveRepoPath()` helper with `realpathAlloc` fallback
  - `worktree/prune.zig` — same pattern for prune and isPrunable operations
  - `worktree/lock.zig` — same pattern for lock, unlock, and isLocked operations
  - `worktree/bare_link.zig` — added `resolvePath()` handling both repo and worktree paths
  - `object/loose.zig` — debug assert enforcing absolute path invariant on `LooseObjectIterator.init()`
- **Leaked Commit allocations in RebasePlanner** (M3) — Added missing `defer commit.deinit()` in `collectRootCommits()` which leaked parents slice, author/committer Identity structs, and message buffer on every iteration of commit traversal (`rebase/planner.zig`)
- **Silent catch continue in tree apply loops** (S1) — Replaced bare `catch continue` with `std.log.warn` including OID and error name in `cli/rebase.zig` and `cli/checkout.zig` tree entry application, making corrupted repos or disk issues diagnosable
- **Leaked username in SSH authenticate** (S2) — Added conditional `defer` in `NativeSshSession.authenticate()` to free the `getDefaultUsername()` allocation when `opts.username` is null; previously leaked on auth method fallback or failure exit paths (`network/ssh_native.zig`)
- **install.sh tar.gz extraction** — Download URL points to `.tar.gz` archive but script was renaming it directly to `hoz` without extracting, causing "exec format error" at runtime. Now properly extracts via `tar -xzf` with temp-dir fallback for varying archive layouts (`bbdee33`)
