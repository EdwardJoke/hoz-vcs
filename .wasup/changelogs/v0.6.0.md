## [v0.6.0] - 2026-05-20

### Added
- **Toon output format** (`64bcc90`) — Native Zig toon formatter with zero external dependencies. ValueBuilder union tree for structured data; renders objects, arrays, and scalars to [toon format](https://github.com/toon-format/toon). `OutputFormat.toon` enum value added; default stays `.human`. Zig 0.16.0 compatible. (`64bcc90`)
- **`--toon` CLI flag** (`a55c47e`) — Global flag sets output format to toon: `hoz --toon <command>`. Parsed before command dispatch so all subcommands inherit it. (`a55c47e`)
- **`hoz --toon status`** (`597461b`) — Structured status output with branch name, staged/unstaged/untracked arrays. Status codes colored by type. (`597461b`)
- **`hoz --toon log`** (`2edd86e`) — Compact commit list with colored hash (yellow), ref (cyan), author (dim), date, and message. Recursive commit walker collects into toon array. (`2edd86e`)
- **`hoz --toon diff`** (`d39604f`) — File change array with change-type (A/M/D/R) and path. (`d39604f`)
- **`hoz --toon branch`** (`d39604f`) — Branch list with name and current marker. (`d39604f`)

### Changed
- **Keyword highlighting** (`64bcc90`) — Terminal ANSI colors applied to key output fields: hash=yellow, ref=cyan, author=dim, branch=green, status codes colored by type (M=yellow, A=green, D=red, R=cyan). Applied in both object fields and array rows. (`64bcc90`)
- **Usage descriptions** (`02f1a74`) — Help text rewritten for all 45+ commands: short, scannable, with one-line descriptions. Key commands now document `--toon` flag. (`02f1a74`)

### Removed
