# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [v0.5.7] - 2026-05-13

### Added
- **Subcommand `--help` / `-h` support** (S1) — All 40+ subcommands now display usage text when invoked with `--help` or `-h` flag. Previously, `hoz add --help` would attempt to read a file named `--help`; now it shows `Usage: hoz add <file> [files...]` (`254a50c`)
- **Log pagination** (`-p` / `--paginate`) — New interactive pagination mode for `hoz log`: displays one commit at a time and waits for Enter key to advance. Skips automatically for `--oneline` mode which is already compact. Usage: `hoz log -p`, `hoz log -p -n 5` (`66fe63a`)
- **`fatalMessage()` method** — New Output module method for critical errors, using `fatal:` prefix with exit code 128, following Git conventions (`baa18e3`)
- **`warningMessage()` method** — New Output module method for non-fatal warnings, using `warning:` prefix, gated behind verbose mode for internal warnings (`baa18e3`)

### Fixed
- **Error message format unification** (M1) — Standardized all CLI error output across 6 modules:
  - `ERROR:` → `error:` lowercase prefix (Git convention)
  - `std.log.err` → `output.errorMessage()` in `cli/add.zig`
  - Inconsistent prefixes in `cli/clone.zig` unified to `fatal:`
  - `{}` → `{s}` format specifier fixes for string arguments
  - Internal warnings now use `output.warningMessage()` gated by `--verbose` in `cli/log.zig`, `cli/rebase.zig`, `cli/checkout.zig` (`baa18e3`)
- **`hoz init` re-run detection** — Running `hoz init` in an existing repository now reports `error: already a Hoz repository in .` instead of silently overwriting the `.git` directory structure (`4cc6130`)
- **Command name leaking into subcommand args** — Root cause fix in `main.zig`: `remaining[arg_offset..]` included the subcommand name as `args[0]`, causing `hoz log` to treat `"log"` as a refspec and show "No commits yet". Fixed by passing `remaining[arg_offset + 1..]`. This affected ALL subcommands — they all previously received their own command name as the first argument (`e794fee`)

### Tests
- **Integration test suite** (M2) — New `tests/integration.sh` with 14 E2E tests covering the full core workflow: init → add → commit → status → log → branch → diff → tag → error handling. Runs against real repo at `~/hozt` (`2690c3d`)
- **Output module unit tests** (S2) — 5 new tests validating error formatting behavior:
  - `errorMessage` format (prefix + args interpolation)
  - `fatalMessage` human format (`fatal:` prefix)
  - `fatalMessage` JSON format (`success:false`, code 128)
  - `warningMessage` format (`warning:` prefix)
  - Lowercase error prefix regression guard (`785c94c`)
