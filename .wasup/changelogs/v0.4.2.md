## [v0.4.2] - 2026-05-09

### Added

#### Installer — `install.sh` (`6d5b194`, `badb580`)

Full POSIX installer script that downloads pre-built binaries from **GitHub Releases**:

| Feature | Details |
|---------|---------|
| **ASCII art banner** | Displays the existing hoz logo on startup |
| **OS detection** | Auto-detects macOS / Linux via `uname -s` |
| **Arch detection** | Maps `x86_64`, `aarch64`, `arm64`, `amd64` to canonical names via `uname -m` |
| **GitHub Release download** | Fetches `hoz-{os}-{arch}` binary from `edwardxie/hoz/releases/download/{tag}/` |
| **Install target** | Defaults to `~/.hoz/bin/hoz`; configurable via `--prefix DIR` |
| **Version pinning** | `--version TAG` installs a specific release; defaults to latest via GitHub API |
| **Existing install check** | Detects prior installation, shows version, prompts before overwrite |
| **chmod +x** | Sets executable permission on downloaded binary |
| **curl / wget fallback** | Tries curl first, falls back to wget; errors if neither found |
| **Shell-aware PATH instructions** | Detects `.bashrc` / `.zshrc` / `.config/fish/config.fish` and prints tailored copy-paste commands |
| **Never auto-modifies shell config** | User must add PATH manually — safe by design |

**Usage:**
```bash
./install.sh                          # Install latest release
./install.sh --version v0.4.2         # Install specific version
./install.sh --prefix /usr/local/bin  # Custom install path
./install.sh --uninstall              # Remove installed binary
./install.sh --help                   # Show all options
```

**Asset naming convention expected in GitHub Releases:**
```
hoz-macos-x86_64    hoz-macos-aarch64
hoz-linux-x86_64    hoz-linux-aarch64
```

#### Uninstaller — `--uninstall` flag (`badb580`)

Removes the binary and cleans up empty install directory:
- Removes `${PREFIX}/hoz`
- Runs `rmdir ${PREFIX}` if empty (no-op if other files exist)
- Prints shell-specific instruction to remove the PATH line from config
- Graceful no-op if hoz is not installed

### Changed

#### Version System — Single Source of Truth (`92b0711`)

**Problem**: Version was hardcoded in 3 locations that could drift out of sync:
| File | Before |
|------|--------|
| [`src/final/final.zig`](src/final/final.zig#L16) | `pub const VERSION = "0.4.2"` (hardcoded) |
| [`src/main.zig`](src/main.zig#L244) | `"hoz version 0.4.2"` (hardcoded string literal) |
| [`build.zig.zon`](build.zig.zon#L17) | `.version = "0.4.2"` (source of truth, but unused at runtime) |

**Solution**: Zig compile-time module options (`std.Build.addOptions`):

1. [`build.zig`](build.zig#L3) declares `const hoz_version = "0.4.2"` as the single source of truth
2. `version_options.addOption([]const u8, "version", hoz_version)` creates a compile-time option
3. Wired into **all 7 module definitions**: exe, mod_tests, exe_tests, integration_tests, fuzz_tests, cross-compile (×2), release (×2)
4. Source files import via `@import("build_options")`:
   - [`final.zig`](src/final/final.zig#L10): `pub const VERSION = build_options.version`
   - [`main.zig`](src/main.zig#L246): `"hoz version " ++ build_options.version`

**Result**: Changing the version now requires editing **only one file** ([`build.zig`](build.zig#L3)). The new value propagates automatically to every binary, test, and cross-compilation target.
